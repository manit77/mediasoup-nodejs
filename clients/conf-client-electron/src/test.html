Kiosk Security Test Page

Use this page to manually verify that the Electron kiosk shell correctly restricts navigation, window opening, and keyboard shortcuts.

1. Navigation restrictions

Try navigating away from the allowed start URL. In a locked-down kiosk, these should be blocked or redirected by the host app / web app.

Expected: the kiosk wrapper or hosted app should ignore / redirect disallowed URLs.

2. Window and popup restrictions

Attempt to open new windows or popups. A kiosk shell usually prevents new top-level windows.

Expected: new windows should not appear as separate OS windows; ideally, they should be blocked or contained by the app.

3. File upload restrictions

Try to select a file from disk. A locked-down kiosk should prevent file uploads.

Expected: the kiosk shell should block file selection so no file name appears and the OS file picker is not useful.

4. Keyboard & shortcut behavior

Use the physical or on-screen keyboard to test restricted shortcuts: Alt+Tab, Cmd/Ctrl+Q, Cmd/Ctrl+W, F11, etc.

Expected: the Electron shell or OS kiosk mode should prevent leaving the app, closing the window, or toggling fullscreen in ways you don't intend.

5. Clipboard & selection

Try selecting text and copying it. In a kiosk, you may want to minimize accidental selection and clipboard leaks.

Tip: This page disables text selection at the HTML level; the hosted kiosk app can do the same if desired.

6. Electron keyboard IPC commands

Call the exposed Electron hooks from the hosted web app and verify that the kiosk shell responds as expected.

Expected: these buttons should call window.electron.* and trigger the corresponding IPC commands in the Electron main process.

7. Form controls test

Use TAB to move between all form controls. The on-screen keyboard and scroll-into-view should work for INPUT, TEXTAREA, SELECT, and BUTTON.

Checkboxes
Radio

Expected: focusing any control (click or TAB) shows the keyboard and scrolls it into view. All controls are pre-filled with test data.

8. Full keyboard input test

Use the on-screen keyboard (simple-keyboard) together with the physical keyboard to exercise all keys, including function keys (F1-F12), Tab, Enter, Backspace, arrows, and symbol keys.

Expected: all characters and navigation keys supported by the kiosk keyboard should appear and behave correctly in this textarea.
